❓ Help & Tutorials
คู่มือการใช้งาน TradeLog Pro+ (Supabase Edition)
🎯 TradeLog Pro+ คืออะไร?
แอปบันทึกการเทรด ใช้ Supabase เป็น Database ที่เชื่อมต่อกับ Google Sheet ผ่าน Apps Script เพื่อ Backup ข้อมูลและ Sync ข้ามอุปกรณ์ได้
✅ ใช้งานได้ทันที
บันทึก Trade, Journal, Data Mode ได้เลยโดยไม่ต้องเชื่อม Sheet — ข้อมูลเก็บใน Browser localStorage อัตโนมัติ
⚡ ขั้นตอนสั้นๆ ในการเชื่อม Google Sheet
1
ตั้งค่า Supabase
เปิด Google Drive → สร้าง Spreadsheet เปล่า → ตั้งชื่อ
›
2
เปิด Apps Script และวางโค้ด
Extensions → Apps Script → วางโค้ดจากแอป → รัน setupSheets()
›
3
Deploy เป็น Web App
Deploy → New Deployment → Web App → Execute as Me → Anyone
›
4
Copy URL → วางใน App
Copy Web App URL → ⚙ → Google Sheet URL → Paste → Save & Test
›
⚠️ ข้อควรรู้
  • ต้องใช้ Google Account — แนะนำ Chrome Browser
  • Apps Script ฟรีทั้งหมด ไม่มีค่าใช้จ่าย
  • Quota: 6 นาที/run, 90 นาที/วัน (เพียงพอสำหรับการใช้งานปกติ)
  • ข้อมูลเก็บใน Supabase ของคุณเอง — ผู้พัฒนาไม่มีสิทธิ์เข้าถึง
TradeLog Pro+ — Supabase Edition
ข้อมูลทั้งหมดเก็บใน Supabase ของคุณ — ปลอดภัย เข้าถึงได้ทุกอุปกรณ์
Login ด้วย Username/Password
กรอก Username และ Password ที่ Admin สร้างให้
Login ด้วย Google
กด "เข้าสู่ระบบด้วย Google" — ระบบจะ redirect ไป Google แล้วกลับมาอัตโนมัติ
ข้อมูลจะ Sync อัตโนมัติ
ทุกการบันทึก Trade จะถูกส่งไป Supabase ทันที
4
เชื่อมต่อ App กับ Google Sheet
ขั้นตอนสุดท้าย — ใช้เวลาไม่ถึง 1 นาที
🔗 วาง URL ใน App
1ใน TradeLog App → คลิก ⚙ มุมบนขวา → "🔗 Google Sheet URL"
2วาง Web App URL ที่ copy มาจาก Apps Script ลงในช่องกรอก URL
3คลิกปุ่ม "💾 Save & Test" — แอปจะทดสอบการเชื่อมต่อทันที
4เชื่อมสำเร็จ → จะเห็น ● Connected มุมบนขวาของ App
☁️ การ Sync ข้อมูล
🔼 Auto Push (App → Sheet)
บันทึก Trade / Journal ทุกครั้งจะ Push ขึ้น Google Sheet อัตโนมัติ
🔽 Manual Pull (Sheet → App)
คลิก Sync หรือ Refresh หน้าเพื่อดึงข้อมูลจาก Sheet มาแสดง
🎉
เชื่อมต่อเสร็จสมบูรณ์!
ข้อมูลทุกอย่าง Sync กับ Google Sheet โดยอัตโนมัติ
สามารถเปิด Sheet ดูข้อมูลได้ทุกเมื่อ หรือแชร์ให้คนอื่น View ได้
🔧 การแก้ปัญหาที่พบบ่อย
❌ เชื่อมต่อแล้วขึ้น Error / "Failed to fetch"▼
  • ตรวจสอบว่า URL ลงท้ายด้วย /exec ไม่ใช่ /dev
  • ตรวจสอบว่าตั้ง "Who has access: Anyone" ตอน Deploy
  • ลอง Deploy ใหม่อีกครั้ง → New deployment
  • ตรวจสอบว่า Google Account ยังล็อกอินอยู่และสิทธิ์ยังไม่หมด
⚠️ หน้าต่าง "This app isn't verified" ปรากฏ▼
เป็นเรื่องปกติสำหรับ Script ส่วนตัวที่ไม่ได้ Submit ให้ Google ตรวจสอบ
วิธีแก้: คลิก "Advanced" → "Go to [Project Name] (unsafe)" → กด Allow
Script ทำงานบน Google Account ของคุณเอง — ปลอดภัย 100%
🔄 ข้อมูลไม่ Sync หรือ Sync ช้ามาก▼
  • Apps Script มี Cold Start — request แรกหลัง idle อาจช้า 3-10 วินาที ปกติ
  • ตรวจสอบ Internet Connection
  • ลอง Refresh หน้าแล้วกด Sync ใหม่
  • ตรวจ Quota: ใน Apps Script → View → Executions
📱 เปิดในมือถือแล้วเชื่อม Sheet ไม่ได้▼
วิธีแก้: เปิด Chrome บนมือถือ → ล็อกอิน Google Account เดียวกันกับที่สร้าง Script → เปิด App ใหม่
App ทำงาน Offline ได้เสมอ — ข้อมูลเก็บใน localStorage แม้ไม่มี Net
🗑️ มี Duplicate rows ใน Sheet▼
ใน App → ⚙ → มีปุ่ม "🧹 Clean Duplicates"
หรือใน Apps Script Editor → รัน Function cleanDuplicates() โดยตรง
💾 ข้อมูลหายหลัง Clear Browser▼
ข้อมูลใน Browser localStorage จะหายถ้า Clear Site Data
ป้องกัน: เชื่อม Google Sheet ไว้เสมอ และใช้ 💾 Save to file เป็น Backup เพิ่มเติม
กู้คืน: กด 📂 Load from file หรือ Sync จาก Sheet ที่เชื่อมไว้
💬 คำถามที่พบบ่อย (FAQ)
